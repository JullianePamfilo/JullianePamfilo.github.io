Software Design and Engineering Artifact 

The improved software design artifact is a testament to my ability to create user-centric, modular applications. I built on top of the initial dashboard program, adding better separation of concerns, clear component responsibilities, and a stronger emphasis on maintainability and usability. This artifact best displays my software architecture principles knowledge and it also helped me to cement the importance of writing code that not only solves the problem at hand, but can also adapt to changing requirements in the future. This artifact also required me to assess an existing system, determine its weaknesses, and make purposeful adjustments to it. This is a critical skill when working on live systems in a professional development environment.

Algorithms and Data Structures Artifact 

The algorithms and data structures artifact represented the transformation from student to competent coder. In the improved course planner application, I swapped out inefficient methods for more suitable data structures such as hash maps and sorted data collections for faster lookups and overall better runtime performance. I also added input validation and normalization to make the application more robust. This artifact is where I think my skills really shined. It required me to think about algorithmic complexity and to be able to justify my design decisions. Optimizing for performance and scalability are key skills for building scalable, enterprise-ready software.

Databases Artifact 

The databases artifact highlights my practical understanding of data persistence, CRUD operations, and the importance of data integrity. In improving the database program using Python, I emphasized validation and error handling, as well as the clear separation between data access and business logic in the application. This artifact shows my ability to handle structured data both responsibly and safely so that applications that depend on the data behave predictably and securely.
